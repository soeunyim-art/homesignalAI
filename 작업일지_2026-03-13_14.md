### 2. 모델 고도화 (predict_model.py)

#### 2-1. 학습/평가 분할 방식 변경

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 방식 | TimeSeriesSplit(5-fold) 교차검증 | 시계열 7:3 고정 분할 |
| 학습셋 | rolling window | 앞 70% (2020-01 ~ 2024-07 내외) |
| 평가셋 | 없음 | 뒤 30% (2024-08 ~ 2026-01 내외) |
| 최종 예측 | - | 전체 데이터 재학습 후 예측 |

이유: 고정 test set 없이는 발표 시 명확한 성능 지표 제시 불가 + 미래 데이터 누수 방지

#### 2-2. 피처 누수 제거 및 정규화

- `avg_price_10k` 피처 제거 → 타겟 변수와 직접 연관, 정보 누수 유발
- `rate_cd`, `rate_bond3y` 제거 → `rate_base`(기준금리)와 중복
- `month` → `month_sin` / `month_cos` 변환 (12월→1월 계절성 순환 인코딩)
- RidgeCV alpha 탐색 범위 확장: `[0.1, 1, 10]` → `[0.01, 0.1, 1, 10, 100, 1000]`

#### 2-3. 아파트별 헤도닉 모델 추가

동별 예측 변화율을 개별 아파트에 적용해 아파트별 예측가 산출.

```
동별 Ridge 모델 → 동별 예측 변화율(%) 산출
                        ↓
            apt_trade 최근 6개월 평균 거래가 조회
                        ↓
         (동별 변화율) × (아파트 현재가) = 아파트별 예측가
```

신규 함수:
- `fetch_apt_current_prices()` — 아파트별 최근 6개월 평균가 조회 (2건+ 거래 기준)
- `predict_apt_level()` — 동별 변화율 → 아파트별 적용
- `save_apt_predictions_to_supabase()` — `predictions_apt` 테이블 저장

출력: `prediction_result_apt.csv`, Supabase `predictions_apt`

### 4. 예측 클리핑 + 신뢰도 점수 (predict_model.py)

#### 4-1. 배경
- 길음동 -30%, 장위동 -38% 등 비현실적 예측 발생
- 원인: 거래량이 적은 동에서 Ridge 모델 과적합

#### 4-2. 예측값 클리핑 (CLIP_BOUNDS)

```python
CLIP_BOUNDS = {1: 10.0, 2: 15.0, 3: 20.0}
# 1개월 예측: ±10% 이내
# 2개월 예측: ±15% 이내
# 3개월 예측: ±20% 이내
```

#### 4-3. 신뢰도 점수 (confidence_score)

```python
def calc_confidence(trade_count: int) -> int:
    return min(100, max(0, int(trade_count * 5)))
# 월 20건 이상 → 100점, 적을수록 낮아짐
```

#### 4-4. Supabase 스키마 업데이트

```sql
ALTER TABLE predictions ADD COLUMN IF NOT EXISTS confidence_score INTEGER;
```
