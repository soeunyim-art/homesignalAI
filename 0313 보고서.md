## 1. 프로젝트 개요 (Executive Summary)

HomeSignal AI는 단순 시계열 통계 예측을 넘어, 부동산 시장에 영향을 미치는 **외부 이슈(뉴스, 개발 호재, 정책 시그널)**를 근거로 방향성을 제시하는 지능형 부동산 분석 서비스입니다. 
**시계열 가격 예측 모델(Prophet+LightGBM)**과 **뉴스 데이터 기반 RAG(Retrieval-Augmented Generation) 챗봇**을 결합하여 부동산 투자자 및 실거주 희망자에게 객관적이고 직관적인 투자 근거를 제시하는 것을 최우선 목표로 합니다.

## 2. 초기 기획 및 아키텍처 설계 (기반 구축기)
> **관련 Docs:** `01_PRD_HomeSignalAI.md`, `02_Architecture_Design.md`, `06_Development_Journal.md` 등 기획 문서 7종

*   **설계의 핵심 원칙:** "FastAPI 기반의 비동기 마이크로서비스 형태" & "명확한 도메인 분리(Forecast, Chat, News)"
*   **핵심 기술 스택 선정:**
    *   **Backend:** Python 3.11+, FastAPI, `uv` (빠른 패키지 의존성 관리)
    *   **Database:** Supabase (PostgreSQL), `pgvector` 기반 벡터 검색, Redis (응답 속도 향상용 캐싱)
    *   **AI / ML:** OpenAI GPT-4o / Claude 3.5 Sonnet 혼용(RAG용), Prophet + LightGBM 앙상블 모델
*   **초기 인프라 스캐폴딩:** 
    *   Supabase 내에 4대 핵심 테이블 설계 완료 (`houses_data` 실거래가, `news_signals` 임베딩, `ai_predictions` 예측 결과, `user_interactions` 로그).
    *   안정적인 예외 처리 전략 (AI API 장애 시 수치만이라도 반환하는 Fallback 설계) 및 공유 모듈(Shared 인터페이스 추상화) 구축.

## 3. 중기 기능 고도화 (데이터 & AI 파이프라인 구축기)
> **관련 Docs:** `03_AI_Model_Pipeline.md`, `IMPLEMENTATION_SUMMARY.md`, `ML_Feature_통합_테이블_가이드.md`

이 시기에는 데이터 분석의 본질인 **상관관계(R² 추이) 극대화**와 피처 엔지니어링에 집중했습니다.
*   **상승 시점 감지 및 키워드 추출 시스템 완성:**
    *   과거 부동산 가격이 실제로 상승했던 시점을 이동평균 교차(MA Crossover) 등으로 자동 감지하여 역산.
    *   상승 트리거가 된 전후의 뉴스 키워드 빈도를 추출하여 가중치로 자동 변환하는 파이프라인 개발 (`RisePointDetector` 도입).
*   **데이터 범위 및 수집 깊이 폭발적 확장:**
    *   기존 '동대문구' 단일 한정 수집에서, 모델 예측 모델 상관관계의 극적 향상을 위해 **'서울 동북권'** 전체로 데이터 범위를 확장 구체화.
    *   크롤링 탐지 키워드를 28개에서 95개로 대폭 확장하고 8개 카테고리(교통, 재개발, 공급 등)로 체계화 (`config/keywords.yaml`).
*   **머신러닝 테이블 구성:** `ml_training_features`로 부동산 가격, 거래량, 뉴스 키워드 빈도, 이벤트 더미를 단일 테이블로 집약 완료.

## 5. 당면 과제 및 주요 한계점 (Issues & Challenges)

1.  **Vercel 서버리스 클라우드 배포 불안정 (500 Internal Server Error)**
    *   **현상:** 로컬 개발 환경(`uvicorn`)에서는 프론트/백엔드 통신 및 DB 구축이 완벽하게 돌아가고 테스트를 통과하나, Vercel 프로덕션 배포 시 `500 Error`가 발생함.
    *   **원인 및 해결 노력:** 모듈의 최상위 Import 에러, 환경변수 누락, 메모리/Time-out 이슈 등을 중점적으로 디버깅하며 배포 안정화를 꾀하고 있습니다.
2.  **데이터 확장에 따른 R² 성능 검증 필요**
    *   동북권 전체로 확대한 데이터가 모델 학습 파이프라인에 온전히 병합된 후, 최종적으로 앙상블 모델 성능(RMSE 하락 및 추세 일치도 향상)이 얼마나 개선되었는지 벤치마크 테스트 및 하이퍼파라미터 튜닝 일정이 대기 중입니다.
