### 실거래 데이터 전체 표시 문제 해결

#### 원인: Supabase PostgREST 1000행 하드캡
- **증상**: 동대문구 매매가 추이 차트가 25.04(4월)까지만 표시되고 이후 데이터 없음
- **원인**: 동대문구 10개 동 × 월별 거래 합산 결과 25.04에서 딱 1000건이 채워져 Supabase 기본 limit에 걸림
- **잘못된 시도**: `.limit(10000)` 코드 추가 → PostgREST max-rows 설정이 1000으로 하드캡이라 코드 limit 무시됨
- **최종 해결**: **동별 개별 REST API 호출 후 합산**
  - 단일 동 1건 = ~97~500행 이내 → 1000행 제한 미도달
  - 10개 동 병렬 조회(`Promise.all`) 후 월별 합산
  - 결과: 동대문구 4개월(25.04까지) → **14개월(25.01~26.02) 전체 표시**

#### 적용 범위
- `app/api/trade-history/route.ts` — 구 단위 조회 전면 재작성
- `app/api/trade-history/apt/route.ts` — 신규 생성 (아파트별 실거래 이력)

#### 구 단위 예측 제거
- **배경**: 구 단위 Ridge 회귀 모델 R² 값이 음수 → 예측 의미 없음
- **변경**: 구(동대문구 등)는 섹션 구분선 헤더로만 사용, 예측값 미표시

- 신뢰도: 거래량 기반 점수 (높음 ≥70 / 보통 40~69 / 낮음 <40)
- 동 클릭 시 실거래 추이 차트 토글 (2025.01~)

- 좌측 사이드바에 "아파트 검색" 메뉴 추가

- 카드마다 현재가 + 1·3·6개월 예측 + 실거래 추이 차트 포함

### 주요 기술 이슈 및 해결

| 이슈 | 원인 | 해결 |
|------|------|------|
| supabase-js `.eq()` 한글 미매칭 | supabase-js 라이브러리 버그 | `.ilike()` 로 대체 |
