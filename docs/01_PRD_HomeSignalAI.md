# HomeSignal AI — 제품 요구사항 명세서 (PRD)

**문서 버전:** 2.0  
**최종 수정일:** 2025-02-25  
**프로젝트:** 동대문구 부동산 시계열 예측 및 RAG 챗봇 서비스

---

## 1. 배경 및 목표 (Background & Goals)

### 배경
동대문구는 청량리 재개발, GTX 호재 등 변동성이 크나, 일반적인 시계열 수치 데이터만으로는 그 이면의 **이슈(이벤트)**를 설명하기 어렵다.

### 목표
단순 수치 예측을 넘어, **과거 뉴스 이슈와 정책적 신호(Signals)**를 결합한 고도화된 부동산 시계열 예측 시스템 및 RAG 챗봇 구축.

---

## 2. 타겟 사용자 (Target Users)

### 주요 사용자
동대문구 지역 **부동산 투자자** 및 **실거주 희망자**.

### 사용자 요구
"단순히 가격이 오를까요?"가 아니라, **"어떤 이슈 때문에 오를 가능성이 높나요?"**에 대한 근거 중심의 답변을 원함.

---

## 3. 핵심 기능 (Core Features)

### 3.1. Must Have (필수 기능)

- **동대문구 시계열 예측 엔진:** 과거 매매가 지수 및 거래량 데이터를 기반으로 한 향후 추세 예측 로직 구현.
- **RAG 기반 전문가 챗봇:** 벡터 DB(타 담당자 영역)와 연동하여 예측 데이터에 대한 설명을 생성하는 로직.
- **백엔드 API 아키텍처:** 프론트엔드와 AI API 사이를 연결하는 안정적인 API 서버.

### 3.2. Should Have (주요 기능)

- **구글 뉴스 크롤링 기반 이슈 분석:** 상승 시점 전후의 주요 이슈(GTX, 재개발 등) 키워드 추출 및 변수화.
- **폴백(Fallback) 처리:** 외부 AI API 오류 시 시계열 예측 수치만이라도 반환하는 예외 처리 로직.

### 3.3. Nice to Have (부가 기능)

- **사용자 피드백 수집:** AI 응답에 대한 좋아요/싫어요 피드백 수집 및 분석.
- **뉴스 감성 분석:** 크롤링된 뉴스의 긍정/부정 척도를 예측 모델의 가중치로 활용.

---

## 4. 성공 지표 (Success Metrics)

### 정량적 지표

- **예측 정확도:** 시계열 모델의 RMSE(평균 제곱근 오차) 최소화.
- **응답 속도:** 전체 RAG 파이프라인 응답 속도 **2.0초 이내** 달성.

### 정성적 지표

- **답변 신뢰성:** 챗봇 답변에 대한 근거 자료(뉴스 출처) 포함 여부.

---

## 6. 개발 프로세스 로드맵 (Roadmap)

| 단계 | 내용 |
|------|------|
| 1. **발견(Discovery)** | 동대문구 부동산 문제 정의 및 가설 설정 |
| 2. **정의(Definition)** | 상세 요구사항 확정 및 PRD 최종화 (본 문서) |
| 3. **설계(Design)** | 시스템 아키텍처 및 프롬프트 설계 |
| 4. **개발(Development)** | 시계열 모델 학습 및 백엔드 로직 구현 |
| 5. **검증(Validation)** | 통합 테스트 및 AI 응답 품질 검증 |
| 6. **출시/운영(Launch/Operation)** | 서비스 배포 및 지속적 개선 |

## 부록: 시계열 모델 평가 프로세스 (Section 03)

코드 작성 시 `.cursorrules` 및 본 PRD와 연계하여 적용한다.

### 표준 파이프라인
```
데이터 추출 → 전처리 → EDA → 학습 → 평가 → 모델 선정
```

### 평가 지표
- **RMSE:** 기본 정확도 지표
- **MAE:** 이상치 영향 완화
- **MAPE:** 상대적 오차 검증
- **방향 정확도:** 상승/하락 추세 예측 정확도
