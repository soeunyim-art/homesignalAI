# 보고일지: 로컬·서버 대시보드 UI·데이터 차이 분석

| 항목 | 내용 |
|------|------|
| 작성 목적 | 로컬 화면(첨부1)과 배포 서버 화면(첨부2) 불일치 원인 기록 |
| 대상 저장소 | HomeSignal AI (`homesignalAI`) |
| 참고 원격 | [github.com/Siyeolryu/homesignalAI](https://github.com/Siyeolryu/homesignalAI) |

---

## 1. 요약

- **첨부2(서버)**는 현재 본 프로젝트의 **Next.js `/dashboard` MVP UI** 및 **FastAPI 예측·뉴스 API**와 구조·수치가 일치하는 편이다.
- **첨부1(로컬)**에 보이는 화면(동북권 평균 매매가·Ridge 회귀 설명·「검색 결과」형 제목·아파트 검색 메뉴 등)은 **이 저장소의 `frontend/app` 소스에서 해당 문구·레이아웃을 찾을 수 없었다.**
- 결론: 차이의 1차 원인은 **“API 응답만 다른 것”이 아니라, 서로 다른 프론트엔드 빌드(또는 다른 프로젝트/브랜치)를 실행 중일 가능성이 매우 높다.**

---

## 2. 첨부2(서버)와 코드베이스 대응 관계

### 2.1 프론트엔드

- 경로: `frontend/app/dashboard/page.tsx`
- 상단 KPI: 예측 지수, AI 예측 신뢰도, 뉴스 기사 수 — `/api/v1/forecast`, `/api/v1/news/insights` 연동
- 사이드 메뉴: 종합 개요 / 지역 분석 / 가격 동향 / AI 리포트 (**4개**, 아파트 검색 항목 없음)
- `frontend/components/dashboard/collapsible-sidebar.tsx`에 메뉴 정의

### 2.2 지도·프라이스맵 영역

- `MapArea` 컴포넌트는 **플레이스홀더**로만 구현됨. 실제 지도/프라이스맵 데이터 연동 없음.
- 서버 스크린샷의 빈 영역 또는 “개발 중” 메시지는 이 구조와 부합한다.

### 2.3 백엔드 예측 수치(예: 106.5, 85%)

- `src/forecast/service.py`의 **`_run_mock_forecast`**: 기준값 105, 스텝마다 +0.5, 기본 `horizon=3`이면 마지막 포인트 **106.5**
- **`_calculate_confidence`**: 현재 구현은 **항상 0.85** → UI **85%**와 일치
- 실제 모델 파일 부재·피처 없음 시 목 데이터로 폴백되며, 차트가 **거의 수평**에 가깝게 보일 수 있음

---

## 3. 첨부1(로컬)과 본 저장소 불일치 사항

다음 요소는 `homesignalAI-main/frontend` 내 검색으로 **존재하지 않음**:

- 「동대문구 **검색 결과** 종합 개요」 유형의 제목
- **동북권 평균 매매가(억 단위)** KPI, 구별 카드(9.5억 등)
- **Ridge regression**, **2020~현재 실거래** 등 문구
- 사이드바 **아파트 검색**, **서울특별시** 필터 등

→ 로컬에서 띄운 앱이 **다른 폴더·옛 브랜치·별도 데모**일 가능성이 크다.

---

## 4. 권장 후속 조치

1. 로컬에서 `npm run dev` / 빌드에 사용한 **디렉터리·브랜치·커밋**과 서버 CI/CD 소스를 동일하게 맞출 것.
2. 서버 배포가 본 저장소 `main` 기준이면 **첨부2 스타일이 정상**이며, 첨부1과 동일 UI를 원하면 해당 소스를 본 저장소에 통합·배포해야 함.
3. 실거래가(억) 기반 화면을 유지하려면 API 스키마·프론트 컴포넌트를 **“예측 지수” 중심 MVP**와 명확히 분리하거나 단일 제품으로 재설계할 것.

---

## 5. (별도 이슈) 화면 수치와 “실제 데이터” 체감 불일치

- 대시보드는 **실거래가가 아닌 예측 지수 시계열**을 사용한다는 안내가 UI에 있음.
- 뉴스 인사이트 일부는 저장소 내 **목/보조 데이터**와 조합된 경로가 있음(`src/news/service.py` 등).
- 지역 파라미터: 프론트는 행정구 **코드(예: 11230)**를 쓰고, API 기본 예시는 구 **이름** 위주로 문서화되어 있어, 환경에 따라 DB 조회 결과가 달라질 수 있음(별도 검증 권장).
