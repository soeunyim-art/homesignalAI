### 1. CORS 및 프로덕션 설정 구현

#### 1.1 Settings에 ALLOWED_ORIGINS 추가

**파일:** `src/config/settings.py`

**변경 내용:**
- `allowed_origins: list[str]` 필드 추가
- `@field_validator` 데코레이터를 사용한 쉼표 구분 문자열 파싱 로직 구현
- `pydantic` import에 `field_validator` 추가

**코드:**
```python
from pydantic import field_validator

class Settings(BaseSettings):
    # CORS 설정
    allowed_origins: list[str] = []

    @field_validator("allowed_origins", mode="before")
    @classmethod
    def parse_allowed_origins(cls, v):
        """쉼표 구분 문자열을 리스트로 파싱"""
        if isinstance(v, str):
            return [x.strip() for x in v.split(",") if x.strip()]
        return v or []
```

**환경변수 사용 예시:**
```bash
ALLOWED_ORIGINS=https://homesignal-ai-frontend.vercel.app,https://custom-domain.com
```

#### 1.2 main.py CORS 미들웨어 로직 개선

**파일:** `src/main.py`

**변경 내용:**
- Debug 모드와 Production 모드 분리
- Production에서 `ALLOWED_ORIGINS` 미설정 시 경고 로그 추가
- CORS 오리진 설정 로직 명확화

**코드:**
```python
# CORS 설정
cors_origins = ["*"] if settings.debug else settings.allowed_origins

if not settings.debug and not cors_origins:
    logger.warning(
        "프로덕션 환경에서 ALLOWED_ORIGINS가 설정되지 않았습니다. "
        "프론트엔드에서 API 호출이 차단될 수 있습니다."
    )

app.add_middleware(
    CORSMiddleware,
    allow_origins=cors_origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**동작 방식:**
- `DEBUG=true` (개발): 모든 오리진 허용 (`["*"]`)
- `DEBUG=false` (프로덕션): `ALLOWED_ORIGINS` 환경변수로 지정된 도메인만 허용
- 프로덕션에서 `ALLOWED_ORIGINS` 미설정 시 경고 로그 출력

## 💡 주요 학습 내용

### 1. Pydantic Field Validator

**Before (Pydantic v1):**
```python
@validator("allowed_origins", pre=True)
def parse_allowed_origins(cls, v):
    # ...
```

**After (Pydantic v2):**
```python
@field_validator("allowed_origins", mode="before")
@classmethod
def parse_allowed_origins(cls, v):
    # ...
```

### 2. CORS 설정 Best Practice

- **개발 환경:** `allow_origins=["*"]` (편의성)
- **프로덕션:** 명시적 도메인 리스트 (보안)
- **환경변수:** 쉼표 구분 문자열로 관리
- **경고 로그:** 설정 누락 시 명확한 경고

### 3. Vercel 배포 전략

- **Monorepo:** 백엔드/프론트엔드 별도 프로젝트
- **Root Directory:** 각 프로젝트의 루트 경로 지정
- **환경변수:** Dashboard에서 설정 후 Redeploy 필수
- **.vercelignore:** 불필요한 파일 제외로 번들 크기 최적화
