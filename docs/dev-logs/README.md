## 📖 개발일지 작성 규칙

### 파일명 규칙
- **형식**: `YYYY-MM-DD.md`
- **예시**: `2026-02-28.md`
- **인코딩**: UTF-8

### 작성 원칙
1. **길게 쓰지 않는다** - 나중에 검색했을 때 이해할 수 있을 정도면 충분
2. **삽질 기록을 빠뜨리지 않는다** - 같은 실수 반복 방지를 위해
3. **"왜(Why)"를 기록한다** - 무엇을 했는지보다 왜 그렇게 했는지
4. **매일 작성한다** - 작은 것이라도 기록하는 습관

## 🔍 검색 팁

### VS Code에서 검색하기
1. `Ctrl + Shift + F` (전체 검색)
2. 검색 위치를 `docs/dev-logs/` 폴더로 지정
3. 키워드나 태그로 검색

### 자주 찾는 내용
- 에러 해결: `#에러` 또는 `🚧 삽질 기록` 검색
- 학습 내용: `💡 새로 알게 된 점` 검색
- 특정 기술: 태그로 검색 (예: `#Supabase`, `#FastAPI`, `#Next.js`)

## 💡 개발일지 활용법

### 1. 주간 회고
- 매주 일요일, 한 주간의 일지를 읽으며 무엇을 배웠는지 정리
- 반복되는 문제가 있다면 해결책 마련

### 2. 월간 리뷰
- 매월 말, 한 달간의 성장을 확인
- 목표 대비 달성도 점검
- 다음 달 계획 수립

### 3. 포트폴리오 자료
- 개발일지는 그 자체로 성장의 증거
- 면접이나 프로젝트 설명 시 활용 가능
