# HomeSignal AI — 프로젝트 발표 보고서

**문서 버전:** 1.0  
**발표일:** 2026-02-28  
**프로젝트:** 동대문구 부동산 시계열 예측 및 RAG 챗봇 서비스  
**발표자:** 백엔드 로직 & RAG 챗봇 담당 개발자

---

## 슬라이드 1. 프로젝트 한 줄 요약

> **"동대문구 부동산, 숫자 예측을 넘어 '왜 오르는지'를 설명하는 AI 서비스"**

HomeSignal AI는 **시계열 가격 예측 엔진**과 **RAG 기반 AI 챗봇**을 결합하여,  
뉴스·정책·개발 호재 등 **근거 중심의 부동산 시장 분석**을 제공합니다.

---

## 슬라이드 2. 프로젝트 배경 — 왜 만들었는가

### 문제 인식

| 기존 서비스의 한계 | HomeSignal AI의 접근 |
|-------------------|---------------------|
| "아파트 가격이 오를까요?" → **숫자만 제시** | "어떤 이슈 때문에 오를 가능성이 높은가?" → **근거 제시** |
| 단순 시계열 모델 (과거 가격 기반) | 뉴스·정책 시그널을 가중치로 반영하는 **하이브리드 모델** |
| AI 답변의 출처 불분명 | RAG로 **뉴스 원문 출처**를 반드시 포함 |

### 왜 동대문구인가

- **청량리 재개발** — 대규모 도시정비 사업 진행 중
- **GTX-C 노선** — 청량리역 광역급행철도 개통 예정
- **이문휘경 뉴타운** — 1만 세대 이상 재개발 추진
- 변동성이 크고 이슈가 풍부하여 **시그널 분석의 효과**를 극대화할 수 있는 지역

## 슬라이드 3. 타겟 사용자

### 주요 사용자 페르소나

| 구분 | 프로필 | 핵심 니즈 |
|------|--------|-----------|
| **투자자** | 30~50대, 동대문구 투자 검토 중 | "지금 사도 되는가? 상승 근거는?" |
| **실거주 희망자** | 결혼·이사를 앞둔 2~3인 가구 | "이 동네 전세/매매 추세가 어떤가?" |
| **시장 관심자** | 부동산 뉴스에 관심 있는 일반인 | "최근 동대문구 이슈가 뭐가 있지?" |

### 사용자의 핵심 질문

```
"동대문구 아파트 가격이 오를까요?"
   → 기존: "향후 3개월 내 2.5% 상승 예측" (끝)
   → HomeSignal AI: "GTX-C 호재(45건)와 재개발 사업 진행에 힘입어
                      상승 가능성이 높습니다. [출처: 한국경제 2024-12-01]"
```

| 계층 | 기술 | 선정 이유 |

## 슬라이드 7. RAG 챗봇 — 핵심 차별점

### RAG(Retrieval-Augmented Generation) 작동 원리

```
사용자: "동대문구 아파트 가격이 오를까요?"
                    │
                    ▼
        ┌───────────────────────┐
        │  1. 질의 임베딩 변환    │  ← text-embedding-3-small (1536차원)
        └───────────┬───────────┘
                    │
                    ▼
        ┌───────────────────────┐
        │  2. Vector DB 유사 검색 │  ← 관련 뉴스 Top-5 검색
        └───────────┬───────────┘
                    │
                    ▼
        ┌───────────────────────┐
        │  3. 시계열 예측 결과    │  ← Forecast API 호출
        │     컨텍스트 결합      │
        └───────────┬───────────┘
                    │
                    ▼
        ┌───────────────────────┐
        │  4. AI 답변 생성       │  ← GPT-4o / Claude 3.5 Sonnet
        │  + 출처 포함           │
        └───────────┬───────────┘
                    │
                    ▼
  "GTX-C 호재(45건)와 재개발 사업 진행에 힘입어
   향후 3개월 내 약 2.5% 상승이 예상됩니다.
   [출처: 한국경제 2024-12-01, 매일경제 2024-11-28]"
```

### 프롬프트 설계 전략

```
[System] 역할 + 톤 + 제약조건
         "당신은 동대문구 부동산 전문 분석가입니다."

[Context] 시계열 예측 JSON + 관련 뉴스 청크 (출처 포함)

[User]   사용자 질문
```

### Fallback (장애 대응)

| 상황 | 대응 |
|------|------|
| AI API 정상 | 예측 수치 + AI 해설 + 출처 |
| AI API 장애 | 예측 수치만 반환 + "일시적 장애" 메시지 |
| 전체 장애 | 표준 에러 응답 + 재시도 안내 |

## 슬라이드 8. API 설계 — 3개 핵심 엔드포인트

### API 엔드포인트 요약

| Method | Endpoint | 기능 | 응답 시간 목표 |
|--------|----------|------|---------------|
| `GET` | `/api/v1/forecast` | 시계열 예측 조회 | 2.0초 이내 |
| `POST` | `/api/v1/forecast` | 상세 예측 요청 | 2.0초 이내 |
| `POST` | `/api/v1/chat` | RAG 챗봇 질의 | 3.0초 이내 |
| `GET` | `/api/v1/news/insights` | 뉴스 키워드 분석 | 1.5초 이내 |

### 예측 API 응답 예시

```json
{
  "region": "동대문구",
  "trend": "상승",
  "confidence": 0.85,
  "forecast": [
    { "date": "2026-03-29", "value": 105.5,
      "lower_bound": 103.5, "upper_bound": 107.5 }
  ],
  "news_weights": [
    { "keyword": "GTX", "frequency": 45, "impact_score": 0.8 },
    { "keyword": "재개발", "frequency": 32, "impact_score": 0.6 }
  ],
  "model_version": "v1.0"
}
```

### 챗봇 API 응답 예시

```json
{
  "answer": "동대문구 아파트 시장은 GTX-C 호재와 재개발 사업 진행에 힘입어...",
  "sources": [
    { "title": "GTX-C 청량리역 개통 예정...",
      "source": "한국경제 2024-12-01", "relevance_score": 0.92 }
  ],
  "forecast_summary": {
    "trend": "상승", "confidence": 0.85
  },
  "fallback": false
}
```

```

│    houses_data       │     │    news_signals      │
│  (부동산 실거래)      │     │  (뉴스/정책 데이터)   │

           │                            │

           ▼                            ▼
┌─────────────────────┐     ┌─────────────────────┐
│   ai_predictions     │     │  user_interactions   │
│  (모델 예측 결과)     │     │  (대화 로그/피드백)   │

└──────────────────────┘     └──────────────────────┘
```

### 주요 기술 요소

- **pgvector 확장** — PostgreSQL에서 1536차원 벡터 유사도 검색 지원
- **GIN 인덱스** — `keywords` 배열 필드 빠른 검색
- **HNSW 인덱스** — `embedding` 벡터 근사 최근접 이웃 검색

## 부록 B. Q&A 예상 질문 및 답변

### Q1. "왜 Fine-tuning이 아니라 RAG를 선택했나요?"

**A:** Fine-tuning은 모델 재학습에 높은 비용과 시간이 필요하고, 뉴스 데이터가 실시간으로 갱신되는 환경에서는 지속적인 재학습이 비효율적입니다. RAG는 검색 데이터베이스만 갱신하면 최신 뉴스를 즉시 반영할 수 있고, 답변의 출처를 명확하게 추적할 수 있어 신뢰성 확보에 유리합니다.

### Q2. "왜 Prophet + LightGBM 조합인가요?"

**A:** Prophet은 시계열의 트렌드와 계절성 분석에 강하지만, 뉴스/정책 같은 외부 이벤트 학습에는 한계가 있습니다. LightGBM은 이런 외부 피처(뉴스 키워드 빈도, 정책 더미 변수)를 학습하는 데 뛰어납니다. 두 모델의 장점을 결합한 앙상블로 예측 정확도를 높일 수 있습니다.

### Q3. "현재 40%인데, 나머지 60%는 얼마나 걸릴까요?"

**A:** 가장 설계가 까다로운 기획·아키텍처 단계가 완료되었고, 백엔드 골격이 모두 잡혀 있어 남은 작업은 '설계를 실제 코드와 서비스로 전환하는 구현' 단계입니다. 데이터 수집(2주) → ML 학습(2~3주) → 서비스 연동(1~2주) → 테스트(1주) → 배포(1주) 순으로 약 **8~10주** 예상됩니다.

### Q4. "동대문구만 가능한 건가요?"

**A:** 현재는 동대문구로 범위를 한정했지만, 국토교통부 API는 전국 데이터를 제공합니다. 뉴스 크롤링 키워드와 DB 필터만 변경하면 타 지역으로 확장할 수 있도록 설계했습니다.

### Q5. "AI API 비용은 어떻게 관리하나요?"

**A:** 세 가지 전략을 적용합니다.
1. **캐싱** — 동일 요청에 대해 Redis 캐시 (예측 1시간, 챗봇 30분 TTL)로 중복 호출 방지
2. **Provider 전환** — 환경변수로 GPT-4o ↔ Claude 전환, 비용 효율적인 모델 선택
3. **호출 상한** — 일일/월간 API 호출 상한 설정으로 예산 초과 방지
