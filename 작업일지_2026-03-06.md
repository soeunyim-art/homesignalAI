# HomeSignal AI — 작업일지

> 작성일: 2026-03-06
> 목표: 동대문구 아파트 매매가 예측 MVP — 예측 모델 개발 및 자동화 파이프라인 완성

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목적 | 동대문구 아파트 매매가 1~3개월 후 예측 |
| 분석 방법 | Ridge 회귀 (전월세가 변동률 + 금리 시차 → 매매가 변화율) |
| 예측 단위 | 동(洞)별 월간 |
| 수집 범위 | 2020-01 ~ 현재 (동대문구 10개 동) |
| 데이터 저장소 | Supabase (PostgreSQL) |

## 3. 이번 세션 완료 항목

### 3-1. Supabase SQL 뷰 생성 (`supabase_views.sql`)

| 뷰 이름 | 설명 |
|---------|------|
| `v_monthly_trade` | 동별 월간 매매 평균가 집계 |
| `v_monthly_jeonse` | 동별 월간 전세 평균 보증금 집계 |
| `v_monthly_wolse` | 동별 월간 월세 평균 집계 |
| `v_model_features` | 위 3개 + 금리 JOIN, 모델 피처 통합 뷰 |

### 3-2. 예측 모델 개발 (`predict_model.py`)

**피처 전략 (24개)**

| 피처 그룹 | 항목 |
|-----------|------|
| 전세 시계열 | avg_jeonse_10k, lag1/lag2/lag3, MoM/YoY 변화율 |
| 월세 시계열 | avg_wolse_10k, lag1/lag2/lag3, MoM/YoY 변화율 |
| 금리 시계열 | base_rate, cd_rate, bond3y, MoM/YoY 변화율 |
| 파생 피처 | 전세가율 (jeonse/trade), 계절성 (month_sin, month_cos) |
| 범주형 | dong (OneHotEncoding, 10개 동) |

**모델 설정**

| 항목 | 값 |
|------|-----|
| 알고리즘 | RidgeCV |
| alpha 후보 | [0.1, 1, 5, 10, 50, 100, 500] |
| 교차검증 | TimeSeriesSplit (5-fold) |
| 타겟 | 동별 avg_price_10k 변화율 (%) |
| 예측 horizon | 1개월 / 2개월 / 3개월 후 |

**출력 파일**

| 파일 | 내용 |
|------|------|
| `correlation_heatmap.png` | 금리·전월세·매매가 상관관계 히트맵 |
| `importance_1m.png` | 1개월 예측 피처 중요도 |
| `importance_2m.png` | 2개월 예측 피처 중요도 |
| `importance_3m.png` | 3개월 예측 피처 중요도 |
| `actual_vs_pred.png` | 동별 실제가 vs 예측가 비교 차트 |
| `price_trend.png` | 동별 매매가 시계열 트렌드 |
| `prediction_result.csv` | 동별 1/2/3개월 후 예측가 수치 |

### 3-3. Supabase predictions 테이블

```sql
CREATE TABLE predictions (
    id BIGSERIAL PRIMARY KEY,
    run_date DATE NOT NULL,
    dong VARCHAR(50) NOT NULL,
    base_ym VARCHAR(7),
    current_price_10k INT,
    pred_1m_10k INT,
    pred_2m_10k INT,
    pred_3m_10k INT,
    change_1m_pct NUMERIC(6,2),
    change_2m_pct NUMERIC(6,2),
    change_3m_pct NUMERIC(6,2),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(run_date, dong)
);
```

## 5. 트러블슈팅 기록

### 5-1. OneHotEncoder — unknown category '이문동'
- **원인**: train 데이터에 없던 동이 예측 시 등장
- **해결**: `OneHotEncoder(handle_unknown='ignore')` 적용

### 5-2. IndexError — `iloc[-1]` on empty group
- **원인**: 일부 동의 유효 데이터가 0건인 경우 예외 미처리
- **해결**: 동별 수동 반복 + `len(valid) > 0` 체크 추가

### 5-3. R² 음수 (-0.329)
- **원인**: 동별 가격 레벨 차이가 커서 절대가격 예측 시 교란
- **해결**: 타겟을 `(미래가 - 현재가) / 현재가 × 100` 변화율(%)로 변경

### 5-4. UnicodeEncodeError — `█`, `—` 특수문자
- **원인**: Windows 터미널 인코딩과 충돌하는 유니코드 문자 사용
- **해결**: `█` → `#`, `—` → `-` 일괄 치환

### 5-5. ValueError — Feature names must be in same order
- **원인**: 예측 시 피처 컬럼 순서가 학습 시와 불일치
- **해결**: `feature_names_in_` 기준으로 피처 순서 고정, 중복 제거 로직 추가

### 5-6. Supabase 400 — 'base_ym' column not found
- **원인**: predictions 테이블이 구버전 스키마(base_ym 컬럼 없음)로 생성되어 있음
- **해결**: `DROP TABLE IF EXISTS predictions` 후 새 스키마로 재생성

### 5-7. 일부 동의 기준월 이상 (2023-08, 2025-04)
- **원인**: 전세 데이터가 없는 월이 있을 경우 해당 동 데이터가 끊김
- **해결**: `build_features()`에서 전세 데이터 누락 월에 forward fill 적용

## 7. 실행 방법

```bash
# 최초 전체 수집 (2020-01 ~ 전월)
python collect_data.py full

# 월별 증분 업데이트
python collect_data.py update

# 예측 모델 실행 (뷰 생성 선행 필요)
python predict_model.py
```

**실행 전 준비사항**
1. `homesignal.env`에 API 키 입력 (`.env.example` 참고)
2. Supabase SQL Editor에서 `supabase_views.sql` 실행
3. Supabase에서 `predictions` 테이블 생성 (스키마 위 참고)
