# 디자인 리뷰 기준표 (MVP 기준)

# 1️⃣ 문제 정의 정렬성 (Problem Fit)

- [ ]  화면이 “확률 기반 예측”이라는 핵심 가치를 분명히 전달하는가?
- [ ]  과거 조회 서비스와 차별점이 시각적으로 드러나는가?
- [ ]  예측 결과가 화면에서 가장 먼저 인지되는가?
- [ ]  사용자가 “지금 사도 되나?”라는 질문에 답을 얻을 수 있는 구조인가?

---

# 2️⃣ 정보 우선순위 (Information Hierarchy)

## Prediction 화면 기준

- [ ]  예측가가 가장 눈에 잘 보이는 위치에 있는가?
- [ ]  신뢰구간이 반드시 함께 노출되는가?
- [ ]  상승 확률이 명확히 %로 표시되는가?
- [ ]  변동성 등급이 시각적으로 구분되는가?
- [ ]  근거(Driver)가 과도하지 않고 3~5개로 제한되는가?

---

# 3️⃣ 불확실성 표현 적절성 (Risk Transparency)

- [ ]  숫자 하나만 제시하지 않고 “구간”을 제공하는가?
- [ ]  신뢰구간이 차트에서 명확히 구분되는가?
- [ ]  예측 하락 시 과도한 공포 색상(빨강) 사용을 피했는가?
- [ ]  데이터 부족 시 신뢰도 낮음을 명확히 안내하는가?
- [ ]  디스클레이머가 예측 화면에 상시 노출되는가?

---

# 4️⃣ UX 흐름 타당성 (Flow Logic)

- [ ]  랜딩 → 예측 결과까지 3~4클릭 이내인가?
- [ ]  CTA 버튼이 과도하지 않은가?
- [ ]  화면 전환이 논리적 흐름을 따르는가?
- [ ]  뒤로 가기/비교/재탐색이 자연스러운가?

---

# 5️⃣ 가독성 & 시각 명확성

- [ ]  다크 모드 대비가 충분한가?
- [ ]  텍스트 크기 계층이 명확한가?
- [ ]  카드 간 간격이 일정한가? (8px 시스템 준수)
- [ ]  차트 범례가 이해 가능하게 배치되었는가?
- [ ]  단위(억/%) 표기가 일관적인가?

---

# 6️⃣ 컴포넌트 일관성

- [ ]  SummaryCard 스타일이 모든 화면에서 동일한가?
- [ ]  Badge(A/B/C) 색상 규칙이 일관적인가?
- [ ]  Alert 스타일이 통일되어 있는가?
- [ ]  버튼 스타일(Primary/Secondary)이 섞이지 않았는가?
- [ ]  동일 기능은 동일 UI 패턴을 따르는가?

---

# 7️⃣ 정책 & 카피 준수

❌ 금지 문구 체크

- [ ]  “매수 추천” 없음
- [ ]  “확실한 상승” 없음
- [ ]  “지금 사세요” 없음

✔ 허용 표현 확인

- [ ]  “상승 확률”
- [ ]  “예측 범위”
- [ ]  “변동성 확대 구간”

---

# 8️⃣ 데이터 예외 처리

- [ ]  거래량 0건일 때 UI가 깨지지 않는가?
- [ ]  예측값 null일 때 placeholder 처리되는가?
- [ ]  API 실패 시 에러 안내가 자연스러운가?
- [ ]  로딩 상태에서 레이아웃이 유지되는가?

---

# 9️⃣ MVP 범위 유지 여부

- [ ]  과도한 기능이 추가되지 않았는가?
- [ ]  개인화 추천 UI가 들어가 있지 않은가?
- [ ]  실시간 업데이트 UI가 포함되지 않았는가?
- [ ]  복잡한 애니메이션이 없는가?
