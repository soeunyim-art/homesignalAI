## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목적 | 동대문구 아파트 매매가 1~3개월 후 예측 |
| 분석 방법 | 회귀 분석 (전월세가 변동률 + 금리 시차 → 매매가) |
| 예측 단위 | 동(洞)별 월간 |
| 수집 범위 | 2020-01 ~ 현재 (동대문구 전체) |
| 데이터 저장소 | Supabase (PostgreSQL) |

## 7. 트러블슈팅 기록

### 7-1. Supabase Python 클라이언트 설치 실패
- **문제**: `supabase>=2.10` 설치 시 pyiceberg, pyroaring 등 C++ 빌드 도구 요구
- **해결**: `supabase>=2.3.0,<2.10.0` 버전 고정 → 이후 Python 라이브러리 완전 제거, `requests`로 REST API 직접 호출로 전환

### 7-2. Supabase API 키 형식 불일치 (Invalid API key)
- **문제**: Supabase 신규 키 형식 `sb_publishable_...`이 Python 클라이언트에서 거부됨
- **해결**: supabase Python 라이브러리 제거 → `requests`로 PostgREST REST API 직접 호출

### 7-3. 국토교통부 API 연결 거부 (포트 8081)
- **문제**: 구 URL `http://openapi.molit.go.kr:8081/...` 차단
- **해결**: 신규 URL `https://apis.data.go.kr/1613000/...` 으로 변경

### 7-4. API 결과코드 "000" 미인식
- **문제**: 구 API는 성공 코드 `"00"`, 신 API는 `"000"` 반환
- **해결**: `if result_code not in ("00", "000")` 조건으로 수정

### 7-5. Supabase RLS 401 오류
- **문제**: Row Level Security 활성화로 INSERT 차단
- **해결**: SQL Editor에서 `ALTER TABLE ... DISABLE ROW LEVEL SECURITY` 실행

### 7-6. 매매 API 403 Forbidden
- **문제**: `getRTMSDataSvcAptTradeDev` 엔드포인트 접근 불가
- **해결**: `getRTMSDataSvcAptTrade` (Dev 제외 버전) 으로 변경

### 7-7. API 필드명 변경 (한글 → 영문)
- **문제**: 신규 API가 `년/월/법정동/아파트` 대신 `dealYear/dealMonth/umdNm/aptNm` 반환
- **해결**: 모든 필드에 영문/한글 이중 fallback 적용
  ```python
  it.get("dealYear", it.get("년", 0))
  ```

### 7-8. 계약기간 형식 오류 (`'24.12~26.12'`)
- **문제**: `contract_term` 값이 정수 대신 날짜 범위 문자열로 수신
- **해결**: `_parse_contract_term()` 함수 추가 — 정수형과 날짜범위형 모두 개월 수로 변환

### 7-9. ON CONFLICT 중복 오류
- **문제**: 동일 배치 내 중복 row 존재 (API가 동일 거래를 복수 반환)
- **해결**:
  - 전처리 단계에서 `area` float → `round(val, 2)` 정규화 후 dict key dedup
  - `Prefer: resolution=ignore-duplicates` 헤더 사용

### 7-10. ECOS 금리 데이터 1건만 수신
- **문제**: CD금리·국고채(3년)는 일별(D) 시리즈인데 월별(M) 쿼리 사용
- **해결**:
  - 일별 시리즈는 전체 기간 일자 조회 후 Python에서 월평균 계산
  - ECOS item code 오류 수정: CD금리 `010502000`, 국고채3년 `010200000`

### 7-11. 코픽스 데이터 없음
- **문제**: ECOS stat_code `921Y001` 데이터 미수록
- **해결**: 코픽스 제거, 3종(기준금리, CD금리, 국고채3년)으로 확정

### 7-12. 누락 월 발생
- **문제**: 전체 수집 후 2024-08, 2025-04 (매매), 2025-05 (전월세) 누락 확인
- **해결**: 해당 월 개별 재수집
