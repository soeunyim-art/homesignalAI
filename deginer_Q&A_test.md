## 0. 테스트 준비

- [ ]  테스트 환경 URL 확인 (dev/stage/prod)
- [ ]  브라우저: Chrome / Safari / Edge 최소 1개씩 확인
- [ ]  해상도: Desktop(1440) / Laptop(1280) / Mobile(375) 확인(반응형 범위 포함 시)
- [ ]  샘플 단지 2개 선정 (거래 데이터 **많은 단지 1**, **적은 단지 1**)
- [ ]  예측 결과 케이스 확보 (**상승 예측 1**, **하락 예측 1**)

---

## 1. 랜딩 페이지 (Landing)

### 1-1. 콘텐츠/CTA

- [ ]  헤더(로고/메뉴/로그인)가 깨지지 않고 표시된다
- [ ]  핵심 메시지(확률 기반 예측, 3개월, 신뢰구간)가 명확히 보인다
- [ ]  Primary CTA(“동대문구 단지 보기”) 클릭 시 **List 페이지로 이동**한다
- [ ]  Feature 카드 3개(예측/리스크/확률)가 노출된다

### 1-2. 정책/문구

- [ ]  디스클레이머가 페이지 하단에 노출된다
- [ ]  과장/투자 권유성 문구가 없다(“추천/확실/지금 사세요” 등)

---

## 2. 단지 리스트 페이지 (Apartment List)

### 2-1. 기본 렌더링

- [ ]  단지 카드 리스트가 정상 로딩된다
- [ ]  카드에 다음 정보가 모두 보인다: 단지명 / 현재 평균가 / 최근 3개월 변동률 / 최근 거래량 / “상세 보기”
- [ ]  리스트 로딩 중에는 Skeleton 또는 로딩 UI가 표시된다

### 2-2. 검색/필터

- [ ]  검색창에 단지명을 입력하면 결과가 필터링된다
- [ ]  검색어 삭제 시 전체 리스트로 돌아온다
- [ ]  (있다면) 지역 드롭다운 변경 시 리스트가 갱신된다
- [ ]  (있다면) 필터(면적/준공/거래량 등) 적용/해제가 정상 작동한다

### 2-3. 예외 케이스

- [ ]  검색 결과가 없을 때 “결과 없음” 안내가 보인다
- [ ]  API 실패 시 Error Alert + 재시도 UI가 나온다

---

## 3. 단지 상세 페이지 (Apartment Detail)

### 3-1. 핵심 정보

- [ ]  단지명/동 정보가 정상 표시된다
- [ ]  요약 카드(현재 평균가/전세가율/최근 변동률/거래량)가 정상 표시된다

### 3-2. 차트/토글

- [ ]  실거래가 차트 영역이 렌더링된다
- [ ]  거래량(막대) 레이어가 있는 경우 함께 표시된다
- [ ]  기간 토글(1Y/3Y/5Y) 클릭 시 차트 데이터가 변경된다
- [ ]  툴팁(날짜/가격)이 정상 노출된다(가격 포맷 포함)

### 3-3. CTA/전환

- [ ]  “3개월 예측 보기” 클릭 시 Prediction 페이지로 이동한다
- [ ]  “비교하기”(있다면) 클릭 시 Compare 페이지로 이동한다

### 3-4. 예외 케이스

- [ ]  거래 데이터 공백 구간이 있을 때 안내 문구가 표시된다
- [ ]  데이터 부족 시 “예측 신뢰도 낮음” 안내가 Detail에서도 노출된다(정책에 따라)

---

## 4. 예측 결과 페이지 (Prediction Result) **핵심**

### 4-1. 요약 카드(4개)

- [ ]  예측가(3개월)가 표시된다
- [ ]  신뢰구간(min~max)이 표시된다
- [ ]  상승 확률(%)이 표시된다
- [ ]  변동성 등급(A/B/C)이 표시된다
- [ ]  수치 단위(억/만원 등)가 일관되다

### 4-2. 예측 차트

- [ ]  과거(실선) + 예측(점선) + 신뢰구간(음영)이 모두 표시된다
- [ ]  범례(실선/점선/음영)가 명확히 표시된다
- [ ]  툴팁에서 날짜/가격이 정상 포맷으로 표시된다
- [ ]  신뢰구간이 과도하게 넓을 때 UI가 깨지지 않는다(레이블 겹침 방지)

### 4-3. 예측 근거(Accordion)

- [ ]  “예측 근거 보기” 기본 닫힘 상태
- [ ]  클릭 시 펼쳐지고, 다시 클릭 시 접힌다
- [ ]  Top driver 3~5개가 표시된다(모멘텀/거래량/전세가율/금리 등)
- [ ]  “모델 신뢰도 등급”과 “최근 평균 오차(±%)”가 표시된다(있다면)

### 4-4. 디스클레이머/카피 준수

- [ ]  페이지 하단에 디스클레이머가 **상시 노출**된다
    
    > 본 서비스는 투자 권유가 아니며 정보 제공 목적입니다.
    > 
- [ ]  금지 문구(매수추천/확실상승/지금사세요)가 없다
- [ ]  예측이 하락일 경우에도 공포 조장 문구 없이 “변동성 확대 구간” 톤으로 표기된다
- [ ]  빨강 색상 과다 사용이 없다(하락=Warning 톤 우선)

### 4-5. 액션 버튼

- [ ]  “비교하기” 클릭 시 Compare로 이동한다
- [ ]  “즐겨찾기” 클릭 시 상태 변화(저장됨)가 보인다(있다면)
- [ ]  “리포트 저장(PDF)” 클릭 시 다운로드/생성 상태가 정상 동작한다(있다면)

### 4-6. 예외 케이스(중요)

- [ ]  예측 API 실패 시 Error Alert + Retry 버튼이 노출된다
- [ ]  데이터 부족 시 신뢰도 C / 안내 문구 / 제한된 예측 제공(정책대로) 된다
- [ ]  확률/신뢰구간 값이 null일 때 UI가 깨지지 않고 placeholder 처리된다(“—”)

---

## 5. 비교 페이지 (Compare)

### 5-1. 선택/전환

- [ ]  단지 A 선택 드롭다운 정상 동작
- [ ]  단지 B 선택 드롭다운 정상 동작
- [ ]  동일 단지 선택 시 안내가 뜬다(또는 선택 불가 처리)

### 5-2. 비교 테이블

- [ ]  현재가/예측가/상승확률/신뢰구간 폭/변동성 등급이 모두 표시된다
- [ ]  값 비교가 읽기 쉬운 정렬/서식으로 제공된다
- [ ]  데이터가 한쪽만 없을 때도 레이아웃이 유지된다

### 5-3. 리스크 차트(옵션)

- [ ]  차트가 있으면 렌더링/툴팁 정상
- [ ]  없으면 placeholder 텍스트라도 제공

### 5-4. 정책/문구

- [ ]  비교 결과에도 “추천/매수” 뉘앙스가 없다
- [ ]  디스클레이머가 노출된다

---

## 6. 공통 UI/UX 품질 (전 페이지)

### 6-1. 내비게이션/상태

- [ ]  뒤로가기/새로고침 시 화면이 정상 복원된다
- [ ]  로딩 시 Skeleton이 과도하게 깜빡이지 않는다
- [ ]  에러 메시지가 기술용어 없이 사용자 친화적이다

### 6-2. 접근성/가독성

- [ ]  주요 텍스트 대비가 충분하다(다크모드 기준)
- [ ]  버튼/링크에 hover/focus 상태가 있다
- [ ]  숫자/단위 표기가 일관된다(억/만원, 콤마)

### 6-3. 성능/안정성

- [ ]  차트 렌더링이 3초 이상 지연되지 않는다(대략 체크)
- [ ]  API가 느릴 때도 UI가 멈추지 않는다

---

## 7. 최종 통과 기준 (Go/No-Go)

- [ ]  예측 결과 화면이 **핵심 정보(예측가/구간/확률/등급)**를 깨짐 없이 보여준다
- [ ]  디스클레이머가 Prediction/Compare에서 상시 노출된다
- [ ]  데이터 부족/에러 케이스에서 UX가 “망가지는” 구간이 없다
- [ ]  금지 문구/투자 권유 뉘앙스가 전 화면에서 없다
- [ ]  3클릭 이내(랜딩→리스트→상세→예측) 핵심 가치 도달이 된다
