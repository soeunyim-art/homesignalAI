# HomeSignal AI — 모델링 종합 보고서

> 작성일: 2026-03-14
> 대상: HomeSignal AI 예측 모델 전체 (동별 Ridge + 아파트별 헤도닉)

---

## 1. 프로젝트 목표

> **"서울 동북권 5개 구의 아파트 매매가를 1~3개월 후 예측한다"**

| 항목 | 내용 |
|------|------|
| 예측 대상 | 동(洞)별 아파트 평균 매매가 변화율 (%) |
| 예측 구간 | 1개월 후 / 2개월 후 / 3개월 후 |
| 분석 지역 | 동대문구 · 성북구 · 중랑구 · 강북구 · 도봉구 (5개 구, 56개 동) |
| 데이터 기간 | 2020-01 ~ 2026-02 (약 6년) |
| 최종 출력 | 동별 예측 변화율 (%) + 예측 매매가 (만원) + 신뢰도 점수 |

---

## 2. 사용 모델 구조

### 2-1. 동별 Ridge 회귀 모델 (메인 모델)

```
[Supabase v_model_features 뷰]
  └── 전월세가 + 금리 + 뉴스시그널 + 계절성 + lag 피처
          ↓
  [StandardScaler + OneHotEncoder (ColumnTransformer)]
          ↓
  [RidgeCV — L2 정규화 선형 회귀]
          ↓
  예측: 동별 1~3개월 후 매매가 변화율 (%)
          ↓
  [CLIP_BOUNDS 클리핑 ± 10/15/20%]
          ↓
  [Supabase predictions 테이블 저장]
```

**Ridge 회귀를 선택한 이유:**
- 피처 간 다중공선성이 높음 (전세가 계열 피처들이 서로 상관)
- L2 정규화가 계수를 0 방향으로 축소 → 과적합 방지
- 데이터 수가 많지 않아 복잡한 비선형 모델보다 안정적

---

### 2-2. 아파트별 헤도닉 모델 (보조 모델)

```
[동별 Ridge 예측 변화율]
          ↓ (동별 변화율 × 아파트 현재가)
  아파트별 1~3개월 예측가 (방식 A)

  병렬로:
[GradientBoostingRegressor]
  입력: 면적 + 층수 + 건물연령 + 동 + 시장상황
  출력: 아파트별 절대 예측가 (방식 B — 헤도닉)
          ↓
  [Supabase predictions_apt 테이블 저장]
```

**헤도닉 모델이 필요한 이유:**
- 동별 Ridge는 "이 동 전체가 +2% 오른다"만 예측
- 같은 동 내에서도 34평 아파트와 16평 아파트의 가격 변동은 다름
- 면적·층수·건물연령 등 물리적 속성 반영을 위해 추가

## 4. 핵심 평가 지표 설명

### R² (결정계수, R-squared)

> "모델이 실제 값의 변동을 얼마나 설명하는가"

| R² 값 | 의미 |
|-------|------|
| 1.0 | 완벽한 예측 |
| 0.5 | 실제 분산의 50% 설명 |
| 0.0 | 평균값으로 예측하는 것과 동일 |
| 음수 | 평균보다도 못한 예측 |

**부동산 도메인 기준:**
- R² ≥ 0.50 : 우수
- R² 0.30~0.49 : 유의미
- R² 0.10~0.29 : 참고 수준
- R² < 0.10 : 무의미

---

### MAE (평균절대오차, Mean Absolute Error)

> "평균적으로 예측값이 실제값에서 얼마나 벗어나는가"

이 모델에서 MAE의 단위는 **% (변화율 기준)**

```
예시: MAE = 13.56%
→ 실제 가격 변화가 +1.5%일 때, 모델 예측은 평균적으로 ±13.56% 빗나감
→ 즉 -12%에서 +15% 사이 어딘가를 예측한다는 뜻
```

**월간 실제 부동산 가격 변화폭이 ±0.5~3% 수준이므로, MAE 13%는 신호보다 오차가 훨씬 큰 상태.**

## 8. 모델 한계 및 솔직한 평가

### 8-1. 현재 모델이 할 수 있는 것
- **구 단위** 매매가 방향성(상승/하락) 시그널 제공
- 전세가율이 높은 지역 = 매매가 상승 가능성 높은 지역 스크리닝
- 신뢰도 배지로 "이 동은 데이터가 부족하니 주의" 경고

### 8-2. 현재 모델이 할 수 없는 것
- **개별 동 수준**의 정확한 가격 예측 (MAE 너무 큼)
- **2개월 이후** 신뢰할 수 있는 예측 (R² 0.18)
- **급격한 정책 변화** 대응 (코로나, 금리 급등 등 구조 변화)

### 8-3. MAE가 높은 근본 원인

```
동별 월 거래 건수: 평균 5~15건
→ 이상치 거래 1건 (예: 신고가/급매) = 평균가 ±10~20% 변동
→ 타겟 변수 자체가 노이즈
→ 모델이 아무리 좋아도 노이즈는 학습 불가
```

## 9. 성능 개선 로드맵

### Phase 1 — 타겟 노이즈 제거 (즉시 가능, 코드 수 줄 수정)

```python
# train_model() 내부에 추가
sub = sub[sub["trade_count"] >= 5]       # 월 5건 미만 동·월 제외
sub = sub[sub[target].abs() <= 15]       # ±15% 초과 이상치 제거
```
**예상 효과:** MAE 13% → 5~7%, R² 소폭 상승

### Phase 2 — 구 단위 집계 학습 (중기)

동 단위(56개, 노이즈 큼) → 구 단위(5개, 월 50~200건)로 집계 후 학습
**예상 효과:** MAE 2~5%, R² 0.5~0.7 가능

### Phase 3 — 뉴스 데이터 축적 후 재통합 (장기)

현재 뉴스 수집 기간이 짧아 대부분 0 → 6개월 이상 축적 후 뉴스 피처 재평가

### Phase 4 — 지역 확장 (Phase 1~2 완료 후)

은평구(6.5억), 서대문구(7.2억) 추가 검토
노원구(5.3억)는 가격 레벨 차이로 보류
